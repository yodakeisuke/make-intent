# インタビュー型ヒアリングパターン

## 原則

ユーザーの暗黙知を言語化する。「指示を待つ」のではなく「質問する」アプローチで、ユーザー自身が気づいていない前提・判断基準・制約を引き出す。

## 3つのルール

1. **表面的・自明な質問を避ける**: 「何を作りますか」ではなく「なぜそれが重要ですか」
2. **最大3問ずつ**: 大量質問を防ぎ、思考のラリーを継続
3. **終了条件を設ける**: 質問が延々続くことを防ぐ

## 引き出すべき暗黙知

| カテゴリ | 質問の方向性 |
|----------|-------------|
| 問題の本質 | 「なぜこれを解決したいのか」「解決しないとどうなるか」 |
| 成功の定義 | 「どうなったら成功か」「誰がそれを判断するか」 |
| トレードオフ | 「AとBのどちらを優先するか」「何を犠牲にできるか」 |
| 制約 | 「絶対にやってはいけないことは何か」「リスク許容度は」 |
| コンテキスト | 「他に何が関係するか」「誰が影響を受けるか」 |

## 質問パターン

### 問題の深掘り

- 「〜という理解で合っていますか？」（仮説確認）
- 「〜と〜のどちらに近いですか？」（二択で明確化）
- 「〜が起きたらどうなりますか？」（失敗シナリオ）

### トレードオフの明確化

- 「速度と品質、どちらを優先しますか？」
- 「これを犠牲にしても達成したいことは何ですか？」
- 「最悪のケースとして許容できるのはどこまでですか？」

### 制約の発見

- 「絶対に避けたいことは何ですか？」
- 「これをやったら問題になることは何ですか？」
- 「誰に確認を取る必要がありますか？」

### 自律性の確認

- 「どこまで自動で進めてよいですか？」
- 「確認が必要なのはどのような場合ですか？」
- 「失敗した場合、どれくらい影響がありますか？」

## 対話の流れ

```
1. ユーザーの入力を受け取る
2. 仮説を立てる（「〜という理解で合っていますか？」）
3. 不明点を3問以内で質問
4. 回答をもとに次の仮説を立てる
5. 十分な情報が集まったらIntent作成へ
```

## 終了判断

以下が明確になったらPhase 2へ移行:
- 解決すべき問題と重要性
- 成功の観察可能な状態
- 主要なトレードオフ優先度
- 絶対的な制約
- 自律性の範囲
